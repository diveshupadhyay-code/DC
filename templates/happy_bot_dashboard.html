Initial setup
1
Clone and install

Upload files to your server or Render. Run pip install -r requirements.txt

2
Set environment variables

Copy .env.example → .env. Fill in the three required keys below.

DISCORD_TOKENYour bot token from Discord Developer Portal
GROQ_API_KEYFree API key from console.groq.com
MONGO_URLMongoDB Atlas connection string (free 500 MB tier)
PORT10000 (default, Render uses this for keep-alive)
3
Render config

Build command: pip install -r requirements.txt
Start command: python main.py

4
Set your owner ID

Open utils/helpers.py and update BOT_OWNER_ID = 876629015144828939 to your Discord user ID.

5
First-run server setup

Run ,quicksetup in your server. Then: ,welcome set #channel → ,welcome enable → ,logs set #mod-logs → ,ticket setup

6
Activate premium

As the owner: ,premium add server YOUR_SERVER_ID to enable AI chat and all premium features for a server.

What's premium

AI chat

@mention Happy or reply. Powered by LLaMA 3.3 via Groq. Per-user conversation memory. Hinglish responses. Blocks @everyone/@here.

Global call

Connect two servers' channels together in real-time. Messages relay across servers. Use ,call to queue, ,hangup to end.

VoiceMaster

Members join a trigger VC → private VC auto-created. Deleted when empty. Setup: ,vcsetup.

Button roles

Interactive role buttons. Click to toggle a role. Persistent across restarts. Setup: ,buttonrole @role Label | @role2 Label.

Bump reminder

Auto-reminds 2h after every DISBOARD bump. Setup: ,bumpreminder on in the reminder channel.

Custom status

Add your server's text to the bot's rotating status pool. Setup: ,setstatus Your text here.

Personal prefix

Works across ALL servers with Happy. Setup: ,prefix self !. Remove: ,prefix selfremove.

Who gets premium

Activated per-server or per-user by the bot owner. ,premium add server ID or ,premium add user ID.

How to add a new cog
1
Create the file

Add cogs/mycog.py. Use the template below as a starting point.

2
Register in main.py

Add "cogs.mycog" to the COGS list in main.py. The loader handles the rest.

3
Restart or hot-reload

Redeploy on Render, or run ,reload mycog if you add a reload command to the owner cog.
