Initial setup
1
Install dependencies

Upload the files to your host. Run pip install -r requirements.txt.

2
Set environment variables

Create a .env file next to main.py and fill in the four keys listed below.

DISCORD_TOKENBot token from the Discord Developer Portal
GROQ_API_KEYFree API key from console.groq.com, used by ai_chat.py
MONGO_URLMongoDB Atlas connection string, free 500 MB tier is enough
PORT10000 by default — Render sets this automatically
3
Render config

Build command: pip install -r requirements.txt. Start command: python main.py.

4
Keep the free tier awake

Render's free web services sleep after 15 minutes without traffic. main.py runs a Flask server on a background thread for exactly this reason — point an external monitor such as UptimeRobot or cron-job.org at the root URL every 10 minutes so the process never spins down between commands.

5
Set your owner ID

Open utils/helpers.py and replace BOT_OWNER_ID with your own Discord user ID. This unlocks every owner-only command.

6
First-run server setup

Run ,quicksetup, then ,welcome set #channel → ,welcome enable → ,logs set #mod-logs → ,ticket setup.

7
Turn on premium

As the owner, run ,premium add server YOUR_SERVER_ID to unlock AI chat, VoiceMaster, level roles and every other gated feature for that server.

What premium unlocks

AI chat

@mention Happy or reply to it. Llama 3.3 via Groq, per-user memory, Hinglish replies, blocks @everyone and @here.

Global call

Connect two servers' channels in real time. ,call to queue or connect instantly, ,hangup to end.

VoiceMaster

Join a trigger channel and get a private voice channel, auto-deleted when it empties. Setup with ,vcsetup create.

Button roles

Click-to-toggle role panels that survive a bot restart. Setup with ,buttonrole @role Label | @role2 Label.

Bump reminder

Pings two hours after every DISBOARD bump, with an optional role ping. Setup with ,bumpreminder on.

Custom status

Adds your server's text into the bot's rotating status pool. Setup with ,setstatus <text>.

Color roles & polish

Self-service accent colors, milestone announcements, embed-only and auto-pin channels. Setup with ,color setup or ,serverpolish.

Level roles

Auto-assigns Lvl 1–100 with cumulative channel permissions as members level up. Setup with ,levelroles setup.

Extra perm roles

Ten hand-out roles — Gif, React, Media, Speak and more — for staff to grant one member at a time.

Adding a new cog
1
Create the file

Add cogs/mycog.py. Use the template below as a starting point.

2
Register it in main.py

Add "cogs.mycog" to the COGS list. The loader logs [OK] or [FAIL] for each cog on startup.

3
Redeploy

Push to your host, or add a reload command to the admin cog for hot-reloading during development.

Cog template
"""cogs/mycog.py — short description.""" import discord from discord.ext import commands from utils.helpers import ctx_mod, ctx_admin, ctx_owner from utils.db import some_col class MyCog(commands.Cog): def __init__(self, bot): self.bot = bot @commands.command() @ctx_mod() async def mycommand(self, ctx): await ctx.reply("Hello!") async def setup(bot): await bot.add_cog(MyCog(bot))

Per-server command control

Disable / enable

Admins run ,disable <command> or ,enable <command> to toggle any single command per server. The on_command listener in admin.py checks the list before every invocation. Core commands like settings, help and sync are protected and can't be disabled.

Command aliases

Admins map a custom word to any existing command with ,alias add <alias> <command>. aliases.py intercepts messages that don't match a real command, rewrites the alias to the real one, and re-invokes it — up to 50 aliases per server.

Reference build. Command counts and descriptions reflect the cogs listed in main.py's COGS registry — update this page when you add, rename or retire a cog.
